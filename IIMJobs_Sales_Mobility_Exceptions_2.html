Sales Mobility -- Exception Rules
Simplified framework for algorithmic job-candidate matching · v1.0
Default matching rule
Same node + same industry = match. Cross-industry is a stretch or hard by default -- do not surface unless the candidate has explicitly opted in to cross-industry search, or the node falls under one of the exceptions below.
Exception 1 Nodes that travel freely across all industries
These nodes are industry-agnostic. A candidate in this subclass can be matched to any industry without penalty. Industry is irrelevant for scoring.

Exception 4 Domain-level variations within the same industry
Within the same industry, domain fine-tunes the match. Same industry + same domain = 1.0. Same industry + adjacent domain = 0.8. Same industry + different domain = 0.6. These are the only cases where domain changes the score within an industry.

Algorithm summary -- 4 rules, in order of application
1
Check Exception 2 (locked nodes) first. If the candidate is on a locked node and the job is outside their home industry, score = 0.05. Stop.
2
Check Exception 1 (free nodes). If the candidate is on a free node, ignore industry entirely. Score on node match only.
3
Same industry? Apply Exception 4 (domain score). Same domain = 1.0, adjacent domain = 0.8, different domain = 0.6.
4
Different industry? Check Exception 3 (permeable pairs). If the pair is listed, score = 0.6. Otherwise score = 0.2 (hard -- only surface if candidate opted in).
